# 代码规范

## 一、后端 Python 规范

### 1.1 编码风格

遵循 PEP 8 规范，关键要点：

| 项目 | 规范 |
|------|------|
| 缩进 | 4 个空格 |
| 行宽 | 最大 120 字符 |
| 命名 | 函数/变量 snake_case，类 PascalCase，常量 UPPER_SNAKE_CASE |
| 导入顺序 | 标准库 → 第三方库 → 本地模块（空行分隔） |
| 类型提示 | 函数参数和返回值使用类型注解 |

### 1.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块/文件 | snake_case | `user_management.py` |
| 类 | PascalCase | `class UserManager:` |
| 函数/方法 | snake_case | `def get_user_by_id():` |
| 变量 | snake_case | `user_name = ""` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT = 3` |
| 私有方法/变量 | _前缀 | `def _internal_method():` |

### 1.3 导入规范

```python
# 1. 标准库
import os
import datetime
from typing import List, Optional

# 2. 第三方库
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session

# 3. 本地模块
from app.core.config import settings
from app.core.database import get_db
from app.models.user import User
from app.schemas.user import UserOut
```

### 1.4 函数编写规范

```python
def get_user_by_id(
    user_id: int,
    db: Session = Depends(get_db)
) -> Optional[User]:
    """根据 ID 获取用户
    
    Args:
        user_id: 用户 ID
        db: 数据库会话
        
    Returns:
        用户对象或 None
    """
    user = db.query(User).filter(User.id == user_id).first()
    return user
```

### 1.5 模型定义规范

```python
class User(Base):
    """用户模型"""
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    username = Column(String(64), unique=True, nullable=False, index=True)
    # ... 其他字段

    # 关系
    department = relationship("Department", back_populates="users")
```

## 二、前端 TypeScript / Vue 规范

### 2.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `ProjectList.vue` |
| 组件名 | PascalCase | `defineComponent({ name: 'ProjectList' })` |
| 组合式函数 | use + camelCase | `useTableData()` |

| 类型/接口 | PascalCase | `interface UserInfo` |
| 全局组件 | 前缀 + PascalCase | `AppButton` |

### 2.3 TypeScript 规范

```typescript
// 优先使用 interface 定义对象类型
interface User {
  id: number

// 使用 type 定义联合类型、工具类型等

type FetchUser = (id: number) => Promise<User>

// 泛型
interface ApiResponse<T = any> {
  code: number
  message: string
  data: T
}
```

## 三、API 设计规范

### 3.1 RESTful 风格

| 操作 | 方法 | 路径示例 | 说明 |
|------|------|---------|------|
| 列表 | GET | `/api/v1/projects/list` | 分页查询 |
| 详情 | GET | `/api/v1/projects/{id}` | 获取单个资源 |
| 创建 | POST | `/api/v1/projects/create` | 新建资源 |
| 更新 | PUT | `/api/v1/projects/{id}` | 完整更新 |
| 部分更新 | PATCH | `/api/v1/projects/{id}` | 部分更新（可选） |
| 删除 | DELETE | `/api/v1/projects/{id}` | 删除资源 |

### 3.2 响应格式

统一使用 `ResponseBase` 包装：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 3.3 错误码

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 / 业务校验失败 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 四、数据库规范

### 4.1 表命名

- 使用小写 + 下划线
- 使用复数形式
- 前缀按业务模块

```
users, projects, tasks, materials, bom_headers
```

### 4.2 字段命名

- 使用小写 + 下划线
- 主键统一为 `id`
- 外键以 `_id` 结尾
- 时间字段：`created_at`, `updated_at`
- 布尔字段：`is_xxx`
- 状态字段：`status`

### 4.3 索引规范

- 主键自动索引
- 外键列加索引
- 常用查询条件加索引
- 唯一约束加唯一索引

### 4.4 通用字段

每张表建议包含：

```sql
id              INT PRIMARY KEY AUTO_INCREMENT
created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
```

## 五、Git 提交规范

### 5.1 Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 5.2 Type 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 代码格式调整（不影响功能） |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具类变动 |

### 5.3 示例

```
feat(project): add project progress calculation

- 添加进度自动计算逻辑
- 关联任务完成率到项目进度

Closes #123
```
