# 项目简介与架构

## 一、项目简介

### 1.1 项目背景

工厂研发项目管理系统（Factory R&D PMS）是一套面向制造企业研发部门的轻量化项目管理平台，旨在解决研发过程中项目进度不透明、实验数据散乱、BOM 变更难追溯、样品试产流程混乱等痛点。

### 1.2 项目定位

- **轻量化**：快速部署、易于上手
- **研发导向**：聚焦研发流程，而非通用项目管理
- **模块化**：五大核心模块，按需使用
- **可扩展**：预留 Phase 3/4 持续迭代

### 1.3 核心价值

| 价值维度 | 说明 |
|---------|------|
| 项目透明化 | 项目进度、任务、需求统一管理，进度可视化 |
| 数据标准化 | 实验数据结构化存储，支持 SPC/CPK 分析 |
| 变更可追溯 | BOM 变更全流程记录，ECN/ECR 管理 |
| 质量可视化 | 样品检验、试产良率统计分析 |
| 知识沉淀 | 文档版本管理，知识库沉淀 |

```
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│               API 网关层 (FastAPI)                      │

                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│               业务逻辑层 (Service Layer)                 │
│  ┌─────────┬─────────┬─────────┬─────────┐  │
│  │ 认证授权 │ 项目管理 │ 实验管理 │ BOM 管理 │  │
│  └─────────┴─────────┴─────────┴─────────┘        │
│  ┌─────────┬─────────┬─────────┐             │

                           │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   MySQL 8    │ │   Redis 7   │ │   MinIO      │
│  关系型数据库  │ │  缓存/会话    │ │  对象存储     │
└──────────────┘ └──────────────┘ └──────────────┘
```

### 2.2 架构分层说明

| 层级 | 技术栈 | 职责说明 |
|------|--------|-----------|
| 前端展示层 | Vue 3 + TypeScript + Element Plus | 用户交互、页面渲染、状态管理 |
| API 网关层 | FastAPI + Pydantic | 路由分发、参数校验、认证鉴权 |
| 业务逻辑层 | SQLAlchemy ORM | 业务规则处理、数据操作 |
| 数据存储层 | MySQL + Redis + MinIO | 持久化存储、缓存、文件存储 |

### 2.3 设计原则

1. **前后端分离**：前端通过 RESTful API 交互，独立部署
2. **模块化设计**：各业务模块低耦合，可独立扩展
3. **配置驱动**：通过环境变量配置，支持多环境部署
4. **容器化部署**：Docker + Docker Compose 一键部署
5. **渐进式开发**：分四个 Phase 逐步完善功能

## 三、模块依赖关系

### 3.1 业务模块关系

```
用户与权限模块
    │
    ├── 项目管理模块
    │       ├── 任务管理
    │       └── 需求管理
    │       │
    │       ├── 实验管理模块
    │       │       └── 实验记录/附件
    │       │
    │       ├── BOM 管理模块
    │       │       ├── 物料主数据
    │       │       └── BOM 变更
    │       │
    │       ├── 样品试产模块
    │       │       ├── 样品检验
    │       │       └── 试产工单
    │       │
    │       └── 文档知识模块
    │               ├── 文档版本
    │               ├── 审批流程
    │               └── 知识库
    │
    └── 部门管理
```

### 3.2 数据关联说明

- **项目** 是核心业务实体，其他模块（实验、BOM、样品、文档）都与项目关联
- **用户** 贯穿所有模块，作为操作人、负责人、审批人等角色
- **文档** 可独立存在，也可关联到项目
