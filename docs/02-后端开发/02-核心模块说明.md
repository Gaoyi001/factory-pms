# 核心模块说明

## 一、配置模块 (config.py)

### 1.1 模块概述

[config.py](file:///g:/资料/code/factory-pms/backend/app/core/config.py) 基于 `pydantic-settings` 实现配置管理，支持从环境变量和 `.env` 文件加载配置。

### 1.2 Settings 类

核心配置类 `Settings` 继承自 `BaseSettings`，包含以下配置分组：

| 配置组 | 配置项 | 说明 |
|--------|--------|------|
| 应用配置 | `APP_NAME`, `DEBUG`, `API_V1_PREFIX` | 应用基本信息 |
| 数据库配置 | `MYSQL_HOST`, `MYSQL_PORT`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`, `DATABASE_URL` | MySQL 连接信息 |
| Redis 配置 | `REDIS_HOST`, `REDIS_PORT`, `REDIS_DB`, `REDIS_URL` | Redis 连接信息 |
| JWT 配置 | `SECRET_KEY`, `ALGORITHM`, `ACCESS_TOKEN_EXPIRE_MINUTES` | JWT 认证配置 |
| MinIO 配置 | `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`, `MINIO_BUCKET`, `MINIO_SECURE` | 对象存储配置 |
| CORS 配置 | `CORS_ORIGINS` | 跨域白名单 |

### 1.3 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `get_database_url()` | `-> str` | 构建 MySQL 连接 URL，支持 `DATABASE_URL` 覆盖 |
| `get_redis_url()` | `-> str` | 构建 Redis 连接 URL，支持 `REDIS_URL` 覆盖 |

### 1.4 使用方式

```python
from app.core.config import settings

# 直接访问配置项
print(settings.APP_NAME)

# 使用方法构建连接串
db_url = settings.get_database_url()
```

### 1.5 配置加载优先级

1. **环境变量**（最高优先级）
2. **`.env` 文件**
3. **代码中的默认值**（最低优先级）

## 二、数据库模块 (database.py)

### 2.1 模块概述

[database.py](file:///g:/资料/code/factory-pms/backend/app/core/database.py) 封装 SQLAlchemy 数据库连接，提供 ORM 基础类和会话管理。

### 2.2 核心组件

| 组件 | 类型 | 说明 |
|------|------|------|
| `engine` | `Engine` | SQLAlchemy 引擎，管理连接池 |
| `SessionLocal` | `sessionmaker` | 会话工厂，创建数据库会话 |
| `Base` | `declarative_base` | ORM 模型基类，所有模型继承自它 |
| `get_db()` | Generator | FastAPI 依赖注入函数，提供会话 |

### 2.3 引擎配置

```python
engine = create_engine(
    DATABASE_URL,
    pool_pre_ping=True,      # 连接前检测连接有效性
    pool_recycle=3600,       # 连接回收时间（秒）
    echo=settings.DEBUG,     # DEBUG 模式下输出 SQL
    connect_args={"charset": "utf8mb4"},
)
```

### 2.4 依赖注入使用

```python
from fastapi import Depends
from sqlalchemy.orm import Session
from app.core.database import get_db

@router.get("/items")
def list_items(db: Session = Depends(get_db)):
    items = db.query(Item).all()
    return items
```

## 三、安全模块 (security.py)

### 3.1 模块概述

[security.py](file:///g:/资料/code/factory-pms/backend/app/core/security.py) 提供 JWT 认证、密码哈希、权限校验等安全功能。

### 3.2 密码哈希

基于 `passlib` + `bcrypt` 实现：

| 函数 | 签名 | 说明 |
|------|------|------|
| `get_password_hash(password)` | `-> str` | 生成密码哈希 |
| `verify_password(plain, hashed)` | `-> bool` | 验证密码 |

### 3.3 JWT 令牌

基于 `python-jose` 实现：

| 函数 | 签名 | 说明 |
|------|------|------|
| `create_access_token(data, expires_delta)` | `-> str` | 创建 JWT 访问令牌 |

**令牌 payload 结构：**
```python
{
    "sub": "123",      # 用户 ID (str)
    "role": "admin",   # 用户角色
    "exp": 1234567890  # 过期时间戳
}
```

### 3.4 认证依赖

| 依赖 | 类型 | 说明 |
|------|------|------|
| `oauth2_scheme` | `HTTPBearer` | 从 Authorization 头提取 Bearer token |
| `get_current_user` | 依赖函数 | 验证 token 并返回当前用户对象 |
| `require_role(*roles)` | 高阶函数 | 返回权限校验依赖，检查用户角色 |

### 3.5 权限系统

用户角色定义：

| 角色 | 说明 |
|------|------|
| `admin` | 管理员，拥有所有权限 |
| `manager` | 项目经理，可管理项目/任务 |
| `member` | 普通成员，可查看和执行任务 |
| `viewer` | 查看者，只读权限 |

**使用示例：**

```python
# 需要登录
@router.get("/me")
def get_me(current_user: User = Depends(get_current_user)):
    return current_user

# 需要特定角色
@router.delete("/{id}")
def delete_item(
    id: int,
    _: User = Depends(require_role("admin", "manager"))
):
    # 只有 admin 或 manager 可删除
    pass
```

## 四、初始化脚本 (run.py)

### 4.1 模块概述

[run.py](file:///g:/资料/code/factory-pms/backend/run.py) 是项目的数据库初始化脚本，用于创建数据表和初始化基础数据。

### 4.2 主要函数

| 函数 | 说明 |
|------|------|
| `init_db()` | 创建所有数据表 + 初始化基础数据 |
| `drop_all()` | 危险操作：删除所有数据表（需交互确认） |

### 4.3 初始化内容

1. **创建数据表**
   - 通过 `Base.metadata.create_all(bind=engine)` 创建所有模型对应的表

2. **默认部门**（仅首次）
   - 管理层、研发部、工艺部、质量部、生产部

3. **管理员账号**（仅首次）
   - 用户名：`admin`
   - 密码：`admin123`
   - 邮箱：`admin@factory-pms.com`
   - 角色：`admin`

### 4.4 命令行使用

```bash
# 初始化数据库（首次运行）
python run.py

# 重置数据库（删除后重建，需确认）
python run.py reset
```
