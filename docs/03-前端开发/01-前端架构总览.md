### 1.1 技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue | ^3.4.0 | 前端框架 |
| Vue Router | ^4.3.0 | 路由管理 |
| Pinia | ^2.1.0 | 状态管理 |
| TypeScript | ^5.4.0 | 类型系统 |
| Element Plus | ^2.7.0 | UI 组件库 |
| Axios | ^1.7.0 | HTTP 客户端 |
| Vite | ^5.2.0 | 构建工具 |
| ECharts | ^5.5.0 | 图表库 |
| Day.js | ^1.11.0 | 日期处理 |

## 二、核心模块

### 2.1 入口文件 (main.ts)

应用入口，负责：
- 创建 Vue 应用实例
- 注册路由
- 注册 Element Plus
- 挂载到 DOM

### 2.2 路由模块 (router/index.ts)

[router/index.ts](file:///g:/资料/code/factory-pms/frontend/src/router/index.ts) 基于 Vue Router 4 实现。

**路由结构：**

```
/login                 登录页
/                      布局框架（需登录）
  ├─ /                  首页仪表盘
  ├─ /projects          项目列表
  ├─ /projects/:id      项目详情
  ├─ /experiments       实验列表
  ├─ /bom               BOM 管理
  ├─ /samples           样品试产
  └─ /documents         文档知识
```

**路由守卫：**

- `beforeEach` 全局前置守卫
- 检查 `localStorage` 中的 token
- 未登录访问受保护页面 → 跳转登录页
- 已登录访问登录页 → 跳转首页
- 动态设置页面标题

### 2.3 请求封装 (utils/request.ts)

[request.ts](file:///g:/资料/code/factory-pms/frontend/src/utils/request.ts) 统一封装 Axios 实例。

**配置：**
- Base URL: `import.meta.env.VITE_API_BASE || '/api/v1'`
- 超时时间: 15000ms
- 默认 Content-Type: `application/json`

**请求拦截器：**
- 从 `localStorage` 读取 token
- 自动注入 `Authorization: Bearer <token>` 头

**响应拦截器：**
- 统一处理响应格式（解包 `data` 字段）
- 非 200 状态码自动提示错误
- 401 状态码自动清除 token 并跳转登录页
- 网络错误统一提示

### 2.4 API 层 (api/index.ts)

[api/index.ts](file:///g:/资料/code/factory-pms/frontend/src/api/index.ts) 按模块组织 API 接口函数。

**API 对象：**

| 对象 | 模块 | 示例方法 |
|------|------|---------|
| `authApi` | 认证 | `login()`, `getMe()` |
| `userApi` | 用户 | `list()`, `create()`, `update()`, `departments()` |
| `projectApi` | 项目 | `list()`, `create()`, `get()`, `getTasks()` |
| `experimentApi` | 实验 | `list()`, `create()`, `getRecords()` |
| `bomApi` | BOM | `listMaterials()`, `listBoms()`, `createChange()` |
| `sampleApi` | 样品 | `listSamples()`, `createInspection()`, `listTrials()` |
| `documentApi` | 文档 | `listDocs()`, `uploadVersion()`, `listArticles()` |

### 4.2 页面组件规范

每个业务模块的页面组件遵循以下结构：

```vue

  <div>
    <!-- 工具栏 -->
    <div class="table-toolbar">...</div>
    <!-- 表格/内容区 -->
    <el-table>...</el-table>
    <!-- 分页 -->
    <el-pagination>...</el-pagination>
    <!-- 弹窗 -->
    <el-dialog>...</el-dialog>
  </div>
</template>

<script setup lang="ts">
import { ref, reactive, onMounted } from 'vue'
import { ElMessage } from 'element-plus'
import { xxxApi } from '@/api'

// 响应式状态
const loading = ref(false)
const tableData = reactive({ items: [], total: 0 })

// 数据加载
const load = async () => { ... }

// 初始化
onMounted(load)
</script>
```

## 五、类型系统

### 5.1 类型定义

`types/` 目录下按模块定义 TypeScript 类型，与后端 Pydantic Schema 对应：

| 文件 | 对应模块 |
|------|---------|
| `user.ts` | 用户相关类型 |
| `project.ts` | 项目/任务/需求类型 |
| `experiment.ts` | 实验相关类型 |
| `bom.ts` | 物料/BOM 类型 |
| `sample.ts` | 样品/试产类型 |
| `document.ts` | 文档/知识类型 |

### 5.2 命名规范

- 接口使用 `I` 前缀或直接使用名词（如 `ProjectOut`）
- 请求参数类型：`XxxCreate`, `XxxUpdate`, `XxxQuery`
- 响应数据类型：`XxxOut`, `XxxDetail`
