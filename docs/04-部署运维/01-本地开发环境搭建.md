# 本地开发环境搭建

## 一、环境要求

### 1.1 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Python | >= 3.9 | 后端运行环境 |
| Node.js | >= 18 | 前端构建环境 |
| npm / pnpm / yarn | - | 包管理器 |
| MySQL | >= 8.0 | 数据库 |
| Redis | >= 6.0 | 缓存 |
| Git | - | 版本控制 |

### 1.2 可选

| 软件 | 说明 |
|------|------|
| Docker Desktop | 一键启动 MySQL/Redis/MinIO |
| MinIO | 对象存储服务（开发可暂用本地） |

---

## 二、后端环境搭建

### 2.1 克隆项目

```bash
git clone <repository-url>
cd factory-pms/backend
```

### 2.2 创建虚拟环境（推荐）

**Windows (PowerShell):**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

**Linux / macOS:**
```bash
python -m venv venv
source venv/bin/activate
```

### 2.3 安装依赖

```bash
pip install -r requirements.txt
```

### 2.4 配置环境变量

复制 `.env` 模板文件（从 deploy 目录复制到 backend 目录）：

```bash
cp ../deploy/.env.example .env
```

修改 `.env` 文件，配置本地数据库和 Redis：

```env
# 应用配置
APP_NAME=Factory R&D PMS
DEBUG=True
API_V1_PREFIX=/api/v1

# 数据库配置（本地开发）
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=factory_pms

# Redis 配置（本地开发）
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0

# JWT 配置
SECRET_KEY=your-secret-key-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440

# MinIO 配置（本地开发可暂用）
MINIO_ENDPOINT=127.0.0.1:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=factory-pms
MINIO_SECURE=False

# CORS 配置
CORS_ORIGINS=["http://localhost:5173"]
```

### 2.5 初始化数据库

确保 MySQL 服务已启动，并创建数据库：

```sql
CREATE DATABASE factory_pms DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后执行初始化脚本：

```bash
python run.py
```

初始化内容：
- 创建所有数据表
- 创建默认部门（管理层、研发部、工艺部、质量部、生产部）
- 创建管理员账号

### 2.6 启动后端服务

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

参数说明：
- `--reload`: 代码修改自动重载（开发用）
- `--host 0.0.0.0`: 监听所有地址
- `--port 8000`: 端口号

### 2.7 验证

访问以下地址验证：

| 地址 | 说明 |
|------|------|
| http://localhost:8000/ | 欢迎页面 |
| http://localhost:8000/health | 健康检查 |
| http://localhost:8000/api/v1/docs | Swagger API 文档 |
| http://localhost:8000/api/v1/redoc | ReDoc API 文档 |

### 2.8 默认账号

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | admin |

## 三、前端环境搭建

### 3.1 进入前端目录

```bash
cd factory-pms/frontend
```

### 3.2 安装依赖

```bash
# 使用 npm
npm install

# 或使用 pnpm（推荐）
pnpm install

# 或使用 yarn
yarn
```

### 3.3 配置环境变量

创建 `.env.development` 文件：

```env
VITE_API_BASE=http://localhost:8000/api/v1
```

### 3.4 启动开发服务器

```bash
npm run dev
```

### 3.5 验证

访问 http://localhost:5173

## 四、使用 Docker 启动基础设施

如果不想在本地安装 MySQL/Redis/MinIO，可以使用 Docker 快速启动：

```bash
cd factory-pms/deploy

# 只启动 MySQL + Redis + MinIO
docker-compose up -d mysql redis minio
```

服务地址：

| 服务 | 地址 | 账号 | 密码 |
|------|------|------|------|
| MySQL | localhost:3306 | pms | pms_secret |
| Redis | localhost:6379 | - | - |
| MinIO API | localhost:9000 | minioadmin | minioadmin |
| MinIO Console | localhost:9001 | minioadmin | minioadmin |

然后按 2.3-2.6 步骤启动后端即可。

## 五、常见问题

### 5.1 MySQL 连接失败

- 检查 MySQL 服务是否启动
- 检查端口是否正确（默认 3306）
- 检查用户名密码是否正确
- 检查数据库是否已创建

### 5.2 pip install 失败

- 尝试使用国内镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 升级 pip：`pip install --upgrade pip`

### 5.3 npm install 失败

- 尝试使用国内镜像：`npm config set registry https://registry.npmmirror.com`
- 删除 node_modules 后重新安装

### 5.4 CORS 跨域问题

- 确认后端 `.env` 中的 `CORS_ORIGINS` 包含前端地址
- 开发环境前端通过 Vite 代理（如配置）

### 5.5 端口被占用

- 后端：修改 `uvicorn` 命令的 `--port` 参数
- 前端：修改 `vite.config.ts` 中的端口配置
