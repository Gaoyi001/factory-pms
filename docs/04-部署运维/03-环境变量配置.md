# 环境变量配置

## 一、配置文件

项目使用 `.env` 文件进行环境变量配置，基于 `pydantic-settings` 实现。

**模板文件**: [.env.example](file:///g:/资料/code/factory-pms/deploy/.env.example)

---

## 二、配置项详解

### 2.1 应用配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `APP_NAME` | Factory R&D PMS | 应用名称 |
| `DEBUG` | True | 调试模式（生产环境设为 False） |
| `API_V1_PREFIX` | /api/v1 | API v1 版本前缀 |

### 2.2 数据库配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `MYSQL_HOST` | mysql | MySQL 主机地址 |
| `MYSQL_PORT` | 3306 | MySQL 端口 |
| `MYSQL_USER` | pms | MySQL 用户名 |
| `MYSQL_PASSWORD` | pms_secret | MySQL 密码 |
| `MYSQL_DATABASE` | factory_pms | 数据库名 |
| `DATABASE_URL` | - | （可选）完整数据库 URL，设置后覆盖上面的配置 |

**DATABASE_URL 格式示例：**
```
mysql+pymysql://user:password@host:port/dbname?charset=utf8mb4
```

### 2.3 Redis 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `REDIS_HOST` | redis | Redis 主机地址 |
| `REDIS_PORT` | 6379 | Redis 端口 |
| `REDIS_DB` | 0 | Redis 数据库编号 |
| `REDIS_URL` | - | （可选）完整 Redis URL，设置后覆盖上面的配置 |

**REDIS_URL 格式示例：**
```
redis://host:port/db
```

### 2.4 JWT 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `SECRET_KEY` | change-this-in-production... | JWT 签名密钥（生产环境必须修改） |
| `ALGORITHM` | HS256 | JWT 签名算法 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | 1440 | Token 过期时间（分钟），默认 24 小时 |

**生成安全的 SECRET_KEY：**
```bash
# Python
python -c "import secrets; print(secrets.token_hex(32))"

# 或使用 openssl
openssl rand -hex 32
```

### 2.5 MinIO 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `MINIO_ENDPOINT` | minio:9000 | MinIO 服务地址 |
| `MINIO_ACCESS_KEY` | minioadmin | MinIO 访问密钥 |
| `MINIO_SECRET_KEY` | minioadmin | MinIO 秘密密钥 |
| `MINIO_BUCKET` | factory-pms | 存储桶名称 |
| `MINIO_SECURE` | False | 是否使用 HTTPS |

### 2.6 CORS 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `CORS_ORIGINS` | ["http://localhost:5173", ...] | 允许的跨域来源（JSON 数组格式） |

**示例：**
```env
CORS_ORIGINS=["http://localhost:5173","https://example.com"]
```

### 2.7 前端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `VITE_API_BASE` | /api/v1 | 前端 API 基础地址 |

## 三、配置加载优先级

从高到低：

1. **系统环境变量** - 最高优先级
2. **.env 文件** - 中间优先级
3. **代码默认值** - 最低优先级

## 四、多环境配置

### 4.1 开发环境 (.env)

```env
APP_NAME=Factory R&D PMS (Dev)
DEBUG=True

MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=dev_password
MYSQL_DATABASE=factory_pms_dev

REDIS_HOST=127.0.0.1
REDIS_PORT=6379

CORS_ORIGINS=["http://localhost:5173"]
```

### 4.2 测试环境

创建 `.env.test`：
```env
APP_NAME=Factory R&D PMS (Test)
DEBUG=False
# ... 其他测试环境配置
```

### 4.3 生产环境

创建 `.env.prod`：
```env
APP_NAME=Factory R&D PMS
DEBUG=False

# 使用强密码
MYSQL_PASSWORD=strong_random_password
SECRET_KEY=very_long_random_string

# 只允许生产域名
CORS_ORIGINS=["https://pms.yourcompany.com"]
```

## 五、配置最佳实践

### 5.1 安全

- [ ] **永远不要**将 `.env` 文件提交到版本控制
- [ ] 生产环境必须修改所有默认密码
- [ ] `SECRET_KEY` 使用足够长的随机字符串
- [ ] 定期轮换密钥和密码

### 5.2 版本控制

`.gitignore` 中应包含：
```gitignore
# 环境变量文件
.env
.env.*
!.env.example
```

### 5.3 Docker Compose 环境变量

在 `docker-compose.yml` 中，后端服务通过 `env_file` 加载 `.env` 文件，同时也可以通过 `environment` 覆盖特定变量：

```yaml
backend:
  env_file:
    - ./.env
  environment:
    MYSQL_HOST: mysql  # 覆盖为 Docker 内部服务名
    REDIS_HOST: redis
```

## 六、配置读取方式

### 6.1 后端 Python

```python
from app.core.config import settings

# 直接访问
print(settings.APP_NAME)
print(settings.MYSQL_HOST)

# 使用方法
db_url = settings.get_database_url()
redis_url = settings.get_redis_url()
```

### 6.2 前端 TypeScript

```typescript
// Vite 环境变量（以 VITE_ 开头的变量才会暴露）
const apiBase = import.meta.env.VITE_API_BASE
```
